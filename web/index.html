01 — The idea

What is text diffusion?

Most language models are autoregressive: they write like a typewriter, one token after another, left to right, each new word conditioned only on what already exists. DiffusionGemma works differently. It lays down a fixed canvas of placeholder MASK tokens and repeatedly denoises the whole block at once — the most confident tokens snap into place first, and the rest fill in around them over a handful of passes.

Autoregressive — typewriter

One token at a time, strictly left → right. Earlier tokens can never be revised.

Diffusion — canvas

The entire block resolves together. Confident words commit early and guide the rest.

02 — The mechanism

The canvas & iterative denoising

Generation happens over a fixed-size token canvas — a block of 256 tokens by default. The canvas begins as pure placeholder/MASK tokens and is refined over up to 48 denoising steps. On each pass roughly 15–20 tokens resolve: the lowest-entropy (highest-confidence) tokens are committed first and immediately act as context for everything still unresolved.

When a canvas is fully denoised it is written to the KV cache and the next canvas begins — so the model is block-autoregressive: diffusion inside a block, left-to-right across blocks.

03 — Why it can self-correct

Bidirectional attention & self-correction

While reading your prompt (prefill) attention is causal — standard left-to-right. But while denoising a canvas attention is bidirectional: every position can see both the left and the right context at the same time.

Because it evaluates the whole block on every step, the model can revise its own earlier guesses. A word that looked plausible on step 3 can be overwritten on step 7 once the surrounding tokens commit — mistakes get fixed in real time, instead of being locked in forever like a typewriter.

Attention, two modes
  • Prefill (prompt): causal — each token sees only what precedes it.
  • Denoising (canvas): bidirectional — full left+right context.
  • Result: in-place editing, infilling, and self-repair within a block.
04 — The sampler

The EB sampler, parameter by parameter

DiffusionGemma uses Entropy-Bounded Denoising with Adaptive Stopping (EB). Every control below is exposed in the Playground — so this page and the demo reinforce each other.

max_denoising_steps default 48

The ceiling on how many refinement passes a single canvas may take. More steps → more chances to revise, but more compute. Adaptive stopping often finishes well before the cap.

temperature_start → temperature_end 0.8 → 0.4

Sampling temperature decays linearly over the steps. Early passes stay exploratory (0.8); later passes turn decisive (0.4) to lock the answer in cleanly.

entropy_bound default 0.1

The confidence gate. Only tokens whose predictive entropy falls under the bound are allowed to commit on a pass — lowest-entropy first. Lower bound = stricter, slower, steadier; higher = bolder commits.

adaptive_stop_threshold default 0.005

Early-exit for a canvas. Once the remaining entropy across the block drops below this, denoising stops — no need to spend the full 48 steps when the block is already settled.

canvas_length default 256

The block size in tokens. The whole block is denoised together; bigger blocks see more context per pass, smaller blocks commit faster.

max_new_tokens · top_k · seed

max_new_tokens caps total output (spread across blocks). top_k optionally restricts each choice to the k most likely tokens (0 = off). seed fixes randomness for reproducible runs (empty = random).

05 — Speed & hardware

Fast, because it denoises in parallel

Resolving many tokens per pass instead of one-at-a-time makes DiffusionGemma roughly 4× faster than a comparable autoregressive model on GPU. It is a Mixture-of-Experts model: 26B parameters total, but only ~3.8B active per token (8 of 128 experts).
